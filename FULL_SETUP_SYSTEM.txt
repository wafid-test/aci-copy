PART 3: LOCAL DEVELOPMENT SETUP
==================================================

Step 1: open terminal in project root

PowerShell:
cd C:\Users\User\Desktop\SVP-MAIN\pci-svi\aci-api

Step 2: install backend dependencies

cd backend
npm install

Step 3: create backend env file

Copy:
- backend\.env.example
To:
- backend\.env

PowerShell:
Copy-Item .env.example .env

Step 4: edit backend\.env

Use these values for local setup:

PORT=4000
NODE_ENV=development
APP_NAME=SVP Backend API
CORS_ORIGINS=http://localhost:3000
JWT_ACCESS_SECRET=change_me_access
JWT_REFRESH_SECRET=change_me_refresh
ACCESS_TOKEN_TTL_SECONDS=900
REFRESH_TOKEN_TTL_DAYS=14
COOKIE_SECURE=false
COOKIE_SAMESITE=lax
DATABASE_URL=postgresql://postgres:your-password@switchback.proxy.rlwy.net:39012/railway
DATABASE_PUBLIC_URL=postgresql://postgres:your-password@switchback.proxy.rlwy.net:39012/railway
SVP_BASE_URL=https://svp-international-api.pacc.sa
SVP_LOCALE=en
SVP_FE_APP=legislator
SESSION_ENC_KEY_BASE64=PUT_BASE64_32_BYTE_KEY_HERE

Important:
- Replace DATABASE_URL with your real database URL.
- Replace JWT secrets with strong random strings.
- Replace SESSION_ENC_KEY_BASE64 with a real base64 key.

Step 5: generate Prisma client

npx prisma generate

Step 6: apply Prisma migrations

If you already have migrations:
npx prisma migrate deploy

If this is first local database setup and you are creating schema locally:
npx prisma migrate dev --name init

Step 7: run backend

npm run dev

Expected result:
- Backend starts on http://localhost:4000
- Health URL: http://localhost:4000/health

Step 8: install frontend dependencies

Open a new terminal:
cd C:\Users\User\Desktop\SVP-MAIN\pci-svi\aci-api\frontend
npm install

Step 9: create frontend env file

Copy:
- frontend\.env.example
To:

PowerShell:

Step 10: edit frontend\.env.local

For live frontend with Railway backend:

Step 11: run frontend

==================================================
PART 5: HOW TO GENERATE SECRETS
==================================================

Example JWT secrets:
- long random strings, minimum 32 characters

Example PowerShell command for a session key:
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))

Use the result as:
- SESSION_ENC_KEY_BASE64=your_generated_value

PART 6: BACKEND LIVE DEPLOY ON RAILWAY
==================================================

This repo already contains:
- backend\railway.toml
- backend\start.sh

Current backend Railway config:
- Builder: RAILPACK
- Start command: bash start.sh
- Health check path: /health

Important:
- Railway service Root Directory must be backend
- Do not deploy backend from repo root

Railway setup steps:

1. Push project to GitHub.

2. Open Railway and create a new project.

3. Choose Deploy from GitHub repo.

4. Select this repository.

5. In Railway backend service settings, set:
   Root Directory = backend

6. Railway should use backend\railway.toml automatically.

7. Add backend environment variables in Railway:

NODE_ENV=production
APP_NAME=SVP Backend API
CORS_ORIGINS=https://svp-book.vercel.app
JWT_ACCESS_SECRET=your_strong_access_secret
JWT_REFRESH_SECRET=your_strong_refresh_secret
ACCESS_TOKEN_TTL_SECONDS=900
REFRESH_TOKEN_TTL_DAYS=14
COOKIE_SECURE=true
COOKIE_SAMESITE=none
DATABASE_URL=postgresql://postgres:your-password@postgres.railway.internal:5432/railway
SVP_BASE_URL=https://svp-international-api.pacc.sa
SVP_LOCALE=en
SVP_FE_APP=legislator
SESSION_ENC_KEY_BASE64=your_base64_32_byte_key

Important database rule:
- On Railway backend service, use Railway internal database URL.
- For local machine, use Railway public proxy URL.

Important Railway rule:
- Railway injects PORT automatically.
- Do not manually add RAILWAY_PUBLIC_DOMAIN
- Do not manually add other RAILWAY_* variables

Expected live backend behavior:
- start script runs prisma generate
- start script runs prisma migrate deploy
- API starts
- health check passes at /health

Expected live backend URL format:
- https://aci-api-production.up.railway.app/health

PART 11: COMMON RAILWAY ERRORS
==================================================

Problem: Railway deploys wrong folder
Cause:
- Service root is repo root instead of backend
Fix:
- Set Root Directory to backend

Problem: Backend starts but Railway marks service unhealthy
Cause:
- App listens on localhost only
Fix:
- Backend must listen on process.env.PORT and host 0.0.0.0

Note:
- Current code in backend\src\index.js is already correct for Railway.

Problem: CORS error in browser
Cause:
- CORS_ORIGINS does not match frontend URL
Fix:
- Set exact frontend URL in Railway backend variables

Problem: Database connection fails
Cause:
- Wrong DATABASE_URL
Fix:
- Use internal Railway DB URL in Railway service
- Use public proxy URL on local machine

Problem: Cookie/session not working on live site
Cause:
- COOKIE_SECURE or COOKIE_SAMESITE incorrect
Fix:
- Use:
  COOKIE_SECURE=true
  COOKIE_SAMESITE=none

Problem: Prisma notice appears in logs
Cause:
- Prisma version update message
Fix:
- Ignore for now if deploy is otherwise healthy
